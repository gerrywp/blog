nginx和apache端口80冲突的问题

2017年12月2日
| linux
| 阅读

　　今天在将nodejs程序部署到远程linux服务器上遇到了问题，很长时间才得以解决！

nodejs程序部署到Linux

使用ftp工具将开发包上传到了linux服务器用户目录，然后进入目录执行node命令：

cd ~/www/website/
node app.js & # 后台运行
# app.js 端口地址3000

app.js里面使用的是'0.0.0.0’监听地址，所以理论上可以接受来自任何IP主机的请求。
假设远程Linux服务器ip为：10.120.110.119,事实上，当我们通过本机访问"http://10.120.110.119:3000" 的时候并不能访问，提示拒绝访问！原因是linux服务器开启了防火墙，需要将3000端口加到防火墙的信任之中，操作如下：

sudo /sbin/iptables -I INPUT -p tcp --dport 3000 -j ACCEPT
# 使用 iptables -vnL 
iptables -vnL # 查看当前系统的防火墙规则，已经添加了同意3000端口
# 待考证(网上说要保存，没试过)
/etc/rc.d/init.d/iptables save

说明：0.0.0.0与127.0.0.1的不同。0.0.0.0可以接受来自网络上所有IP的请求，而127.0.0.1只能接受本机的回环地址的请求。所以app.listen的第二个参数要置空，否则外网就只能通过反向代理来访问！

安装nginx反向代理

# 下载包
cd /usr/local/src
curl -O http://nginx.org/download/nginx-1.13.7.tar.gz
# 解压缩
tar -zxvf nginx-1.13.7.tar.gz
cd nginx-1.13.7
# 配置生成Makefile文件
./configure #可使用 --prefix-path=
make install
# 默认安装到/usr/local/nginx路径下
cd /usr/local/nginx/sbin
./nginx -t #测试是否安装成功

上述执行完毕，当运行./nginx -c /usr/local/nginx/conf/nginx.conf启动的时候报80端口被占用。因为服务器上装了apache，80端口被httpd服务占用。

# 查看80端口占用情况
sudo netstat -antp | grep 80

apache的默认安装路径在/www/wdlinux/apache下

# 进入apache配置文件目录
cd /www/wdlinux/apache/conf
# 这里可以设置apache-httpd的端口
vim httpd.conf
# 查看httpd服务状态(占用了80端口)
service httpd status

查看服务器的外网ip地址

# 以下命令可以查看服务器的外网IP地址
curl ifconfig.me
