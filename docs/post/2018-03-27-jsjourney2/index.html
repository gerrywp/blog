　　express-handlebars服务端模板引擎的局部渲染。非SPA应用程序，在最原始的动态页面并使用了layout布局的情况下， 我们通常希望能够将index.hbs里面的UI结构和js逻辑部分组合在一起，并且需要将js逻辑统一放置到layout布局页的最底部。 handlebars的Block Helper能够帮我们解决这个问题！

Block Helper

通常我们的layout.hbs文件结构组织如下:

<!DOCTYPE html>
<html lang="en">
  <head>
    <!--引入css文件-->
    <link href="./*.css"/>
  </head>
  <body>
    <header></header>
    <main>
      { { {body} } }
    </main>
    <footer></footer>
    <!--javascript 脚本-->
    <script  src="./javascripts/jquery.min.js"/>
    <!--其它页面的脚本放置在这里-->
  </body>
</html>

index.hbs文件结构如下：

<div class="container">
  <div class="row">
    <div class="col-12"></div>
  </div>
</div>
<script>
  $(function(){
    //do something here
  })();
</script>

很明显index.hbs页面需要在layout.hbs页面的jquery.js加载之后再加载。也就是在物理顺序上index.hbs页面的js逻辑代码， 要放置在laout.hbs中
